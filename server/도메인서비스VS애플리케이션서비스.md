# 도메인서비스 VS 애플리케이션서비스

## 도메인 서비스 어플리케이션 서비스 차이?

 + 도메인 서비스는 도메인 로직을 가지고 있따.
 + 애플리케이션 서비스는 그렇지 않다.

 + 정리를 해보자면 비즈니스 결정을 도메인 모델에 위임. 
 + 마지막 샘플코드를 보면 if로 예외 처리하고, 코드를 나누며 도메인 모델안에서만 해결하게하고
 + 애플리케이션 서비스는 비즈니스 결정을 내리지않고 코드를 순차적으로 실행만시킨다 이 느낌?
 + 최종적으로 도메인 모델은 격리되어있음.


## 언제 도메인 서비스를 추출하나?

 + 여기서는 외부 자원이있을때 (DB같은) 도메인모델이 위임할 필요 없음 알아서해 
 + 글이 좀어렵게 되있어서 최종적으론 코드를 나눠서 관심사를 분리..
 + 도메인 서비스는 테스트가 가능하다. 단위테스트로 외부 의존성을 줄이자.

## blog

 + 도메인 서비스는 자연스럽지않는 도메인 지식을 제공하고 있찌만 
 + 도메인 서비스를 도입하는 이유는 도메인 모델의 격리와 관련 있음.
 + 도메인 서비스 어플리케이션 서비스는 VO를 상위에 다루는 stateless한 클래스를 의미 함.

### 도메인 서비스

 + 비즈니스 의사결정과 관련된 코드 담당
 + Entity/ VO에 속할 수 없는 도메인 로직 다룰 때 사용

### 도메인서비스 출출

 + 도메인 지식은 도메인  서브스에 위임, 애플리케이션 서비스는 도메인 서비스의 오케스트레이션(관리 담당?) 담

### 도메인 서비스 장점

 + 도메인 서비스 분리시 단위 테스트 작성하기 편함
 + 도메인 지식이 응집 되 있어 가독성이 좋음
